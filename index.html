How this works

Keep one row per repository in a spreadsheet (Excel, Google Sheets, etc.), export it as a CSV, and upload it above. The dashboard groups repos by migration phase for the Daily view, and rolls those phases up into Scheduled / In Progress / Complete for the Monthly view — automatically, based on the table below.

Click Download CSV template for a ready-to-edit starting file with the columns already set up.

Columns

ColumnRequiredAccepted valuesNotes
repo_nameYesAny text Name of the repository being migrated.
migration_phaseYes Preparation In Progress Complete Post-Migration Validation Drives the Daily dashboard columns.
health_statusOptional On Track At Risk Blocked Done Shown in the "Status" section. Defaults to On Track if left blank.
target_dateOptionalYYYY-MM-DD Drives the Date picker and Monthly grouping. Defaults to today if left blank.
waveOptionalAny text e.g. "Wave 1" — shown on each repo card.
ownerOptionalAny text Who's responsible for this repo's migration.

How the Monthly rollup is calculated

The Monthly dashboard's three columns are derived automatically from migration_phase:

Daily phaseMonthly column
PreparationScheduled
In Progress, Post-Migration, ValidationIn Progress
CompleteComplete

Tips

Column names are matched flexibly — "Repo Name", "repo_name" and "Repository" all work, as do "Status", "Health" and "RAG" for health status. If a required column can't be found, the dashboard will leave your current data in place and let you know.
